Runiq RAG upsert pipeline sample document.

This document is intentionally simple and deterministic. It is used to demonstrate the Document to Chunk to Embedding to Vector Store upsert flow in a small developer-facing sample.

The flow starts with a document that has an id, content, and optional metadata. The document content is split into ordered chunks by the configured chunker, and each chunk keeps its relationship with the source document.

Each chunk is then embedded by a deterministic fake embedding provider, so the sample can run without an API key, network access, or a real embedding service. The provider always produces the same small vector for the same chunk content.

The chunk and embedding pairs are handed to the Vector Store upsert pipeline. The pipeline maps them into vector records, validates the vector dimensions against the target index, and writes the records into the in-memory vector store under an explicit index name.

The sample does not call a production vector database and does not demonstrate retrieval or query behavior. Its only purpose is to make the upsert pipeline visible from the console output.

Expected result: the sample should print the index name, the document id, the generated chunk ids, the embedding dimensions per chunk, and the upsert result including attempted, processed, and failed record counts.
