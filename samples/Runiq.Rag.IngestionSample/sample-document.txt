Runiq RAG ingestion sample document.

This document is intentionally simple and deterministic. It is used to demonstrate the Document to Chunk to Embedding flow in a small developer-facing sample.

RAG ingestion starts with a document that has an id, content, and optional metadata. The document content is then split into ordered chunks by the configured chunker. Each chunk keeps its relationship with the source document.

Chunk order matters because retrieval systems often need to preserve the original sequence of information. A chunk should expose its index, content preview, source document id, and metadata values such as start index, end index, and approximate token count.

After chunking, each chunk is prepared as an embedding input. The embedding provider receives the chunk content and returns a vector. In this sample, a deterministic fake embedding provider is used so the output can be verified without an API key or network access.

The sample does not call a production embedding service. It does not write vectors to a vector store. Its only purpose is to make the ingestion pipeline visible from the console output.

Expected result: the sample should print the document id, generated chunk ids, chunk indexes, metadata values, content previews, embedding dimensions, and the relationship between every chunk and its embedding.
